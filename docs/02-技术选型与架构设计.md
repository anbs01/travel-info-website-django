# 技术选型与架构设计

---

## 一、技术选型

| 层次         | 技术                       | 说明                                                     |
| ------------ | -------------------------- | -------------------------------------------------------- |
| 后端框架     | Django 4.x                 | 内置 ORM、路由、Admin、认证，适合内容站                  |
| 数据库       | MySQL 8.x                  | 成熟稳定，中小型项目首选                                 |
| 前端样式     | Tailwind CSS               | 按设计稿还原，响应式灵活，无需切图                       |
| 前端交互     | Alpine.js                  | 轻量，处理弹窗、下拉、Tab 等简单交互，无需引入 Vue/React |
| 富文本编辑器 | django-ckeditor 或 TinyMCE | 后台内容编辑，支持图片/视频上传                          |
| Web 服务器   | Nginx                      | 反向代理、静态文件托管                                   |
| 应用服务器   | Gunicorn                   | Django 生产环境标准搭配                                  |
| 部署环境     | Linux（Ubuntu 22.04 LTS）  | 推荐，稳定、成本低                                       |
| 图片处理     | Pillow                     | 水印叠加、等比缩放                                       |

## 二、项目目录结构

```

│   ├── news/                # 资讯模块
│   └── dashboard/           # 后台管理（基于 Django Admin 定制）
│
├── templates/               # HTML 模板
│   ├── base.html            # 公共基础模板
│   ├── partials/            # 公共片段
│   │   ├── header.html
│   │   ├── footer.html      # 非首页通用底部
│   │   ├── footer_home.html # 首页专用底部
│   │   ├── pagination.html
│   │   └── region_modal.html  # 地区选择弹窗（前后台共用）
│   ├── home/
│   ├── places/
│   ├── travelogue/
│   ├── foods/
│   ├── goods/
│   ├── news/
│   └── static_pages/        # 静态页：404、403、会员登录注册等
│
├── static/                  # 静态资源
│   ├── css/
│   ├── js/
│   └── images/
│       ├── noimage.png
│       ├── watermark.png
│       └── qrcode.png
│
├── media/                   # 用户上传文件
│   └── admin/
│       └── YYYYMM/          # 按月自动切换
│
├── manage.py
└── requirements.txt
```

## 三、应用模块划分

### 3.1 places（城乡）

- 省份 / 国家
- 国内城乡 / 海外城乡
- 打卡点
- 景区
- 交通点

### 3.2 travelogue（攻略）

- 攻略文章

### 3.3 foods（美食）

- 美食条目

### 3.4 goods（文创）

- 文创商品

### 3.5 news（资讯）

- 资讯文章

### 3.6 core（公共）

- 搜索词管理
- 建议 / 揪错 / 补充材料提交
- 版权引用
- 关于我们 / 服务协议 / 合作事宜 / 联系我们
- 首页推荐配置
- 地区选择公共组件

### 3.7 dashboard（后台）

- 基于 Django Admin 定制
- 登录鉴权、30 分钟无操作自动退出
- 数据的增删改查、置顶、屏蔽、移动排序

## 四、URL 路由设计

```
/                              # 首页
/places/                       # 城乡主页
/places/search/                # 城乡搜索结果页
/places/ur/<slug>/             # 城乡详情页（slug 为城乡代码）
/places/scenic<id>/            # 景区详情页
/places/traffic<id>/           # 交通详情页
/travelogue/                   # 攻略主页
/travelogue/<YYYYMMDDHHmm>/    # 攻略详情页
/foods/                        # 美食主页
/foods/f<id>/                  # 美食详情页
/goods/                        # 文创主页
/goods/g<id>/                  # 文创详情页
/news/                         # 资讯主页
/news/<YYYYMMDDHHmm>/          # 资讯详情页
/about/                        # 关于我们
/service/                      # 服务协议
/contact/                      # 联系我们
/cooperation/                  # 合作事宜
/feedback/                     # 意见建议
/search/                       # 公共搜索结果页
/404/                          # 404 页
/403/                          # 403 页
/member/                       # 会员登录/注册（静态页）
/admin/                        # 后台管理（Django Admin）
```

> ID 从 10001 开始，攻略/资讯详情页 URL 中的时间戳在发布时生成并存入数据库，修改内容后不变。

## 五、SEO 方案

- 服务端渲染（SSR），爬虫直接获取完整 HTML
- 每个页面独立的 `<title>` 和 `<meta description>`，由后台数据填充
- 语义化 HTML 标签
- 自动生成 `sitemap.xml`（使用 `django.contrib.sitemaps`）
- `robots.txt` 屏蔽后台路径

## 六、图片处理方案

- 上传时用 Pillow 处理：等比缩放、水印叠加
- 水印位置可选：居中 / 中下 / 右下
- 上传目录按月自动切换：`media/admin/YYYYMM/`
- 列表焦点图：CSS `object-fit: cover` 居中裁剪，不加水印
- 详情页图片：最大宽度 960px，超出等比缩小

## 七、部署架构

```
用户请求
  ↓
Nginx（80/443）
  ├── /static/  → 直接返回静态文件
  ├── /media/   → 直接返回上传文件
  └── 其他      → 转发给 Gunicorn
                    ↓
                 Django 应用
                    ↓
                 MySQL
```

- HTTPS 使用 Let's Encrypt 免费证书
- 推荐服务器：2核2G Linux VPS，初期完全够用

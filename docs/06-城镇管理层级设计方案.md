# 城镇管理与地理层级技术设计规格书

## 一、 设计目标
构建一套支持六级行政区划（国家、省、市、县、乡、村）的统一地理信息系统，并在此基础上关联美食、景区、文创、交通等业务模块。

---

## 二、 核心地理模型：`tp_geo` (统一地理层级表)

### 2.1 数据库结构
使用**单表自关联树**结构，存储所有行政层级。

| 字段名 | 类型 | 说明 |
| :--- | :--- | :--- |
| **Id** | int | 主键，自增 |
| **ParentId** | int? | 父级 ID（国家级为 NULL） |
| **Level** | int | 层级标识：1=国家, 2=省, 3=市, 4=县, 5=乡镇, 6=村 |
| **Title** | varchar | 节点名称（如：山东省、烟台市） |
| **Slug** | varchar | 拼音/英文代码，用于 URL（如：yantai） |
| **Nature** | varchar | 性质：Domestic(国内), Overseas(海外) |
| **AncestorPath** | varchar | 祖先路径，用于穿透查询。格式如 "1/10/20" |
| **IsHidden** | bool | 是否隐藏 |
| **SortOrder** | int | 排序权重 |

### 2.2 核心查询逻辑
*   **获取某省下所有城镇**：`WHERE AncestorPath LIKE '1/10/%'` (假设 1/10 是中国/山东的路径)。
*   **获取直接子级**：`WHERE ParentId = @Id`。

---

## 三、 交通点模型：`tp_transports` (独立业务表)

交通点不属于地理行政层级，而是附着在地理层级上的**业务内容**，与美食、文创、景区并列。

### 3.1 数据库结构
继承 `BaseContent` 公共基类。

| 字段名 | 类型 | 说明 |
| :--- | :--- | :--- |
| **Id** | int | 主键，自增 |
| **GeoId** | int | **外键**，关联到 `tp_geo` 的 ID (通常关联到市/县/镇级) |
| **TransportType** | varchar | 类型：Airport(机场), Railway(火车站), Bus(汽车站), Pier(码头) |
| **Address** | varchar | 详细地址 |
| **ContactPhone** | varchar | 联系电话 |
| **Longitude/Latitude**| decimal | 经纬度坐标 |

---

## 四、 业务关联规范

所有业务模块（美食、景区、文创、游记、交通点）均通过统一的 `GeoId` 字段与地理层级关联。

*   **关联规则**：业务内容可以关联到 `tp_geo` 的**任意层级**。
*   **现有字段调整**：
    *   `tp_foods.RegionId` → 重命名/迁移为 `GeoId`
    *   `tp_scenic_spots.RegionId` → 重命名/迁移为 `GeoId`
    *   `tp_creative_products` 增加 `GeoId`

---

## 五、 数据迁移与清理路径

### 5.1 废弃旧表
必须彻底废弃现有的分表设计，消除冗余：
1.  **[DELETE]** `tp_regions` (原省级表)
2.  **[DELETE]** `tp_places` (原城镇表)

### 5.2 迁移步骤
1.  **创建新表**：初始化 `tp_geo` 和 `tp_transports`。
2.  **导入层级数据**：
    *   将 `tp_regions` 数据导入 `tp_geo` (Level=2)。
    *   将 `tp_places` 数据导入 `tp_geo` (Level=3~6，根据原 PlaceType 映射)。
3.  **修复业务外键**：将 `tp_foods` 等表中的旧外键 ID 映射到 `tp_geo` 的新 ID。

## 六、 前端展示逻辑

| 页面 | 查询策略 |
| :--- | :--- |
| **省份/城乡列表** | 通过 `Nature` 和 `Level` 筛选顶层节点，通过 `AncestorPath` 筛选下级。 |
| **城镇详情页** | 根据当前 `GeoId` 并行查询各个业务表：<br>`SELECT * FROM [业务表] WHERE GeoId = @currentGeoId` |
| **城乡搜索页** | 基于 `tp_geo` 的层级关系进行 Tab 切换查询。 |

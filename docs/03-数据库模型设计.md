# 数据库模型设计

本文档定义各功能模块的核心数据库模型结构，基于 Django ORM 规范，采用抽象基类减少冗余。

---

## 一、公共抽象基类

### 1.1 BaseContent
资讯类内容（攻略、资讯、美食、文创）及城乡、景区的公共父类。

| 字段名 | 类型 | 说明 |
| :--- | :--- | :--- |
| title | CharField(200) | 简称/短标题，用于列表、版面受限处 |
| full_title | CharField(200) | 全称，用于详情页及版面宽敞处；若为空则回退显示 title |
| summary | TextField | 简要概述 / 特色说明 |
| image | ImageField | 焦点图，允许为空（资讯、攻略可无图） |
| image_show_watermark | BooleanField | 详情页图片是否显示水印，默认 False |
| watermark_pos | CharField | 水印位置：center / bottom_center / bottom_right |
| redirect_url | URLField | 跳转链接，填写后前台点击直接跳转，不进详情页 |
| views | PositiveIntegerField | 阅读量，默认 0 |
| is_sticky | BooleanField | 是否置顶，默认 False |
| sticky_at | DateTimeField | 置顶时间，用于多置顶排序，置顶时自动写入 |
| is_home | BooleanField | 是否推荐至首页/主页，默认 False |
| is_hidden | BooleanField | 是否屏蔽（保留数据但前台不显示），默认 False |
| created_at | DateTimeField | 添加时间，auto_now_add |
| updated_at | DateTimeField | 修改时间，auto_now |

> `watermark_pos` 仅在 `image_show_watermark=True` 时生效。
> `full_title` 为空时前台自动回退到 `title`，无需在视图层重复判断。

## 二、地区与城乡模块（places app）

### 2.1 Region（省份 / 国家）

| 字段名 | 类型 | 说明 |
| :--- | :--- | :--- |
| name | CharField(100) | 简称 |
| full_name | CharField(200) | 全称，用于详情页面包屑和弹窗 |
| introduction | TextField | 介绍内容，用于省份弹窗 |
| is_overseas | BooleanField | 是否海外，默认 False |
| sort_order | PositiveIntegerField | 排序权重，默认 0，支持移动操作 |
| created_at | DateTimeField | auto_now_add |

> 国内时 `name` 为省份简称（如"浙江"），`full_name` 为全称（如"浙江省"）。
> 海外时 `name` 为国家简称，`full_name` 为国家全称。

### 2.2 Place（城乡）
继承 `BaseContent`。

| 字段名 | 类型 | 说明 |
| :--- | :--- | :--- |
| region | ForeignKey(Region) | 所属省份/国家，on_delete=PROTECT |
| english_code | SlugField(unique=True) | 纯英文代码，构成 URL：`/places/ur/<english_code>/` |
| place_type | CharField | 地级市 / 县级市 / 乡镇 / 村庄 |
| is_overseas | BooleanField | 是否海外，默认 False |
| hide_nav | BooleanField | 是否隐藏详情页导航栏，默认 False |
| sort_order | PositiveIntegerField | 排序权重，支持移动操作，默认按 created_at 顺序 |

> `english_code` 加唯一约束，一旦创建不允许修改，保障 URL 永久稳定。
> ID 在应用层从 10001 起分配（Django AutoField 默认从 1 起，通过初始 migration 设置 AUTO_INCREMENT=10001）。

### 2.3 ScenicSpot（景区 / 打卡点）
继承 `BaseContent`。

| 字段名 | 类型 | 说明 |
| :--- | :--- | :--- |
| place | ForeignKey(Place) | 所属城乡，on_delete=CASCADE |
| spot_type | CharField | scenic（景区）/ checkin（打卡点） |

> URL 规则：景区 `/places/scenic<id>/`，打卡点复用景区详情页模板。
> 两者共用一张表，通过 `spot_type` 区分，避免模型冗余。

### 2.4 Traffic（交通出行）
继承 `BaseContent`。

| 字段名 | 类型 | 说明 |
| :--- | :--- | :--- |
| place | ForeignKey(Place) | 所属城乡，on_delete=CASCADE |
| traffic_type | CharField | 机场 / 火车站 / 汽车站 / 水上交通 / 轨道交通 / 地面交通 |

> 前台显示顺序：按 `traffic_type` 固定权重排序，同类型内按 `created_at` 升序。
> 权重映射在代码常量中定义，不存入数据库。
> URL：`/places/traffic<id>/`

## 三、资讯类频道模块

### 3.1 News（资讯）
继承 `BaseContent`。

| 字段名 | 类型 | 说明 |
| :--- | :--- | :--- |
| slug | CharField(20, unique=True) | 发布时生成（YYYYMMDDHHmm），构成 URL，发布后不可改 |
| source | CharField(100) | 来源，允许为空 |
| author | CharField(100) | 作者，允许为空 |
| content | TextField | 富文本正文 |
| place | ForeignKey(Place, null=True) | 关联城乡，可选 |

### 3.2 Travelogue（攻略）
继承 `BaseContent`。

| 字段名 | 类型 | 说明 |
| :--- | :--- | :--- |
| slug | CharField(20, unique=True) | 同 News.slug 规则 |
| source | CharField(100) | 来源，允许为空 |
| author | CharField(100) | 作者，允许为空 |
| content | TextField | 富文本正文 |
| place | ForeignKey(Place, null=True) | 关联城乡，可选 |

### 3.3 Food（美食）
继承 `BaseContent`。

| 字段名 | 类型 | 说明 |
| :--- | :--- | :--- |
| content | TextField | 富文本正文 |
| place | ForeignKey(Place, null=True) | 关联城乡，可选 |
| source | CharField(100) | 来源，允许为空 |

### 3.4 Goods（文创 / 商品）
继承 `BaseContent`。

| 字段名 | 类型 | 说明 |
| :--- | :--- | :--- |
| content | TextField | 富文本正文 |
| place | ForeignKey(Place, null=True) | 关联城乡，可选 |
| source | CharField(100) | 来源，允许为空 |

## 四、公共功能模块（core app）

### 4.1 HomeTextRecommend（首页文字推荐）
独立于各内容模型，由管理员单独配置。

| 字段名 | 类型 | 说明 |
| :--- | :--- | :--- |
| title | CharField(200) | 推荐标题 |
| url | URLField | 跳转链接 |
| is_active | BooleanField | 是否启用，不删除可随时关闭 |
| sort_order | PositiveIntegerField | 排序 |

> 此类推荐不显示日期，始终置于文字列表最顶部。

### 4.2 SearchKeyword（搜索词）

| 字段名 | 类型 | 说明 |
| :--- | :--- | :--- |
| keyword | CharField(50) | 关键词 |
| sort_order | PositiveIntegerField | 排序 |
| is_active | BooleanField | 是否启用 |

### 4.3 SiteInfo（网站基本信息）
单例模型，存储关于我们、服务协议、联系我们等页面内容。

| 字段名 | 类型 | 说明 |
| :--- | :--- | :--- |
| about | TextField | 关于我们 |
| service | TextField | 服务协议 |
| contact | TextField | 联系我们 |
| cooperation | TextField | 合作事宜 |
| copyright | TextField | 版权信息（底部） |
| copyright_home | TextField | 首页专用版权信息 |

### 4.4 Feedback（反馈提交）

| 字段名 | 类型 | 说明 |
| :--- | :--- | :--- |
| feedback_type | CharField | suggestion（建议）/ correction（揪错）/ supplement（补充材料） |
| target_model | CharField(50) | 关联内容的模型名，如 "news"、"place" |
| target_id | PositiveIntegerField | 关联内容的 ID |
| content | TextField | 提交内容 |
| extra | TextField | 补充材料专用附加说明，允许为空 |
| user_ip | GenericIPAddressField | 提交者 IP |
| created_at | DateTimeField | auto_now_add |

> 不使用 GenericForeignKey，避免查询复杂度，用 `target_model + target_id` 字符串组合定位即可。

## 五、排序与查询规则汇总

### 5.1 默认排序

| 模块 | 排序规则 |
| :--- | :--- |
| 资讯、攻略、美食、文创 | 置顶优先（sticky_at 倒序），其余按 created_at 倒序 |
| 城乡、省份、景区、交通 | 置顶优先，其余按 sort_order 升序（移动操作修改此值） |
| 交通（前台） | traffic_type 固定权重升序，同类型内 created_at 升序 |

### 5.2 屏蔽过滤
所有前台查询统一加 `.filter(is_hidden=False)`，建议封装为自定义 Manager：

```python
class PublicManager(models.Manager):
    def get_queryset(self):
        return super().get_queryset().filter(is_hidden=False)
```

### 5.3 slug 生成规则
News / Travelogue 发布时自动生成，若同一分钟内有重复则追加秒数后两位：

```
默认：YYYYMMDDHHmm        → 202603210845
冲突：YYYYMMDDHHmmSS      → 20260321084532
```

### 5.4 ID 起始值
美食、文创、景区、交通的 ID 从 10001 开始，在对应 app 的初始 migration 中设置：

```python
migrations.RunSQL("ALTER TABLE foods_food AUTO_INCREMENT = 10001;")
```

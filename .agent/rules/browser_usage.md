# 浏览器工具使用规范 (Browser Usage Rules)

## 1. 核心原则
严禁在未经用户明确授权的情况下主动打开或读取浏览器页面。Token 节约与源码分析能力优先。

## 2. 强制流程
在调用 `browser_subagent` 或 `read_browser_page` 前，必须执行以下步骤：
1. **深度源码复核**：必须至少尝试两次不同维度的源码排查（如：模板解析、后端逻辑、CSS 定义分析）。
2. **风险/收益说明**：向用户明确解释为什么要动用浏览器（例如：怀疑环境层面的静默错误）。
3. **获取授权**：必须在得到用户明确的“同意”、“OK”或“打开吧”等指令后方可执行浏览器相关工具。

## 3. 免责场景
* 仅当用户主动要求“截图看看”、“帮我录制演示”或“去某个网站爬取数据”时，可直接执行。
* 除上述场景外，所有 UI 调试必须先假设是源码逻辑问题。
